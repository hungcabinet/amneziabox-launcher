# What's New from 0.8.2

## Version 0.8.3 Release Notes

<details>
<summary><b>🇷🇺 Русская версия / Russian Version</b></summary>

## ✨ Основное

**Парсер и конфиг (WireGuard, endpoints)**

- **Парсер wireguard://** — ссылки поддерживаются как прямые. Полный разбор URI: приватный ключ из userinfo, query-параметры (publickey, address, allowedips, mtu, keepalive, presharedkey, listenport и др.); `listen_port` только при ненулевом значении; плюс в ключах сохраняется. Ноды WireGuard попадают в секцию endpoints (sing-box 1.11+).
- **Конфиг endpoints** — секция для нод WireGuard. В OutboundGenerationResult добавлен EndpointsJSON; в модель визарда — GeneratedEndpoints и OutboundStats.EndpointsCount. Сборка конфига записывает сгенерированные endpoints между маркерами @ParserSTART_E / @ParserEND_E; updater при обновлении из подписок пишет и outbounds, и endpoints. В шаблон добавлена секция `"endpoints": []`.
- **Документация ParserConfig** — в ParserConfig.md описаны WireGuard в connections, формат wireguard://, маркеры endpoints и требование sing-box 1.11+.

**Шаблон визарда (wizard_template.json)**

- **Шаблон** — почищен, убрано лишнее; TUN на macOS возвращён (параметры darwin-tun для чекбокса TUN на Mac).

**Конфиг (загрузка)**

- **Clash API и имена с # и //** — исправлено падение Clash API на названиях с `#` и `//`. Удалены самописные stripComments и removeTrailingCommas; api/clash.go и core/config/config_loader.go используют только jsonc.ToJSON для нормализации JSON; использование regexp в этих путях убрано.

**Визард (UX и поведение)**

- **Кеш превью** — сбрасывается при изменении источников или ParserConfig (добавление/удаление источника, префикс, конфигуратор, ручной JSON, загрузка состояния).
- **View по источнику** — окно использует кеш превью при наличии (мгновенно после Refresh).
- **Get free VPN** — диалог подтверждения перед применением («текущие источники и правила будут заменены»).
- **Диалог Add/Edit Outbound** — при переключении с вкладки Raw на Settings поля формы (тег, тип, комментарий, фильтры и т.д.) подтягиваются из введённого JSON.
- **Конфигуратор аутбаундов** — применение изменений обновляет поле ParserConfig синхронно, чтобы при переключении вкладки модель не перезаписывалась устаревшим текстом.
- **Scope при редактировании** — при редактировании аутбаунда Scope можно менять; при сохранении аутбаунд переносится из старого scope (global или источник) в новый.

**Визард (модель как источник истины)**

- **Конфигуратор и диалог** — больше не принимают ParserConfig по ссылке; данные берутся из модели (presenter.Model()). Коллбэки списка источников и вкладки Outbounds читают модель в момент вызова.
- **UIUpdater и Model()** — в интерфейс добавлен метод Model(); бизнес-слой принимает UIUpdater и получает модель из него. Тип ModelUpdater удалён.
- **Префикс тега** — общий числовой индекс (1:, 2:, 3:, …) для подписок и connection-only блоков. При Append новые прокси получают индексы после существующих.
- **Сборка списка прокси** — один тип (proxyInput) и одна функция (buildProxiesFromInputs) для подписок и блока connections; createSubscriptionProxies и matchOrCreateConnectionProxy удалены.
- **Parser** — удалён мёртвый код (countConnectionOnlyProxies); введена isConnectionOnlyProxy(); лог «Not preserving other connection ProxySources» только в режиме Apply.

**Диагностика (STUN)**

- **Настройки STUN** — кнопка ⚙ рядом с «STUN [UDP]»: свой STUN-сервер (host:port), на macOS галочка «Use system SOCKS5 proxy» (через прокси или напрямую). Кнопка «?» у поля ввода открывает список публичных STUN-серверов [always-online-stun](https://github.com/pradt2/always-online-stun?tab=readme-ov-file#always-online-stun-servers). Кнопка STUN на всю ширину, шестерёнка справа; в диалоге поле ввода на всю ширину, «?» справа.

**Diagnostics и Help**

- **Вкладка Diagnostics** — кнопка «Open logs» разделена на две: «Open log window» (иконка окна) и «Open logs folder» (иконка папки). Повторное нажатие «Open log window» передаёт фокус уже открытому окну логов, не создавая дубликат.
- **Вкладка Help** — кнопка «Open Logs Folder» убрана (открытие папки с логами доступно с Diagnostics).

**Исправления**

- **Исчезновение источников** — исправлено после повторного открытия визарда, смены префиксов и сохранения. ParseAndPreview больше не подставляет поле URL в ParserConfig; превью и сохранение используют только ParserConfig.Proxies как источник истины. Для запуска парсинга превью поле URL больше не обязательно.
- **Документация/комментарии** — уточнено: ParserConfig является источником истины для источников; SourceURLs используется только как ввод для кнопки Add.

## ✨ Highlights

**Parser & config (WireGuard, endpoints)**

- **Parser wireguard://** — Links supported as direct links. Full URI parsing: private key from userinfo, query params (publickey, address, allowedips, mtu, keepalive, presharedkey, listenport, etc.); `listen_port` only when non-zero; plus sign in keys preserved in query. WireGuard nodes are emitted into the endpoints section (sing-box 1.11+).
- **Config endpoints** — Section for WireGuard nodes. OutboundGenerationResult has EndpointsJSON; wizard model has GeneratedEndpoints and OutboundStats.EndpointsCount. Config build writes generated endpoints between markers @ParserSTART_E / @ParserEND_E; updater writes both outbounds and endpoints when updating from subscriptions. Template includes `"endpoints": []`.
- **ParserConfig docs** — ParserConfig.md documents WireGuard in connections, wireguard:// format, endpoints markers, and sing-box 1.11+ requirement.

**Wizard template (wizard_template.json)**

- **Template** — Cleaned up, redundant content removed; TUN on macOS restored (darwin-tun params for TUN checkbox on Mac).

**Config (loading)**

- **Clash API and names with # and //** — Fixed Clash API failing on names containing `#` or `//`. Removed homemade stripComments and removeTrailingCommas; api/clash.go and core/config/config_loader.go now use only jsonc.ToJSON for normalizing JSON; regexp usage dropped in these paths.

**Wizard (UX & behaviour)**

- **Preview cache** — Invalidated when sources or ParserConfig change (Add/Del source, prefix, configurator apply, manual JSON, load state).
- **View for a source** — Window uses the existing preview cache when available (instant after Refresh).
- **Get free VPN** — Confirmation dialog before applying ("Applying this configuration will replace your current sources and rules. Continue?").
- **Add/Edit Outbound dialog** — When switching from Raw tab to Settings tab, form fields (tag, type, comment, filters, etc.) are now filled from the raw JSON.
- **Outbounds configurator** — Applying changes (Add/Edit/Delete) updates the ParserConfig text field synchronously so that switching to another tab does not overwrite the model with stale content.
- **Scope when editing** — When editing an outbound, Scope can now be changed; the outbound is moved from the old scope (global or source) to the new one on Save.

**Wizard (model as source of truth)**

- **Configurator and dialog** — No longer take ParserConfig by reference; they read from the model (presenter.Model()). Sources list and Outbounds tab callbacks read the model at call time.
- **UIUpdater and Model()** — Interface now includes Model(); business layer takes UIUpdater and gets the model from it. ModelUpdater type removed.
- **Tag prefix** — Single numeric index (1:, 2:, 3:, …) shared across subscriptions and connection-only blocks. On Append, new proxies get indices continuing after existing ones.
- **Proxy list building** — One type (proxyInput) and one function (buildProxiesFromInputs) for both subscriptions and connection block; createSubscriptionProxies and matchOrCreateConnectionProxy removed.
- **Parser** — Dead code removed (countConnectionOnlyProxies); isConnectionOnlyProxy() used for connection-only detection; "Not preserving other connection ProxySources" log only in Apply mode.

**Diagnostics (STUN)**

- **STUN settings** — ⚙ button next to "STUN [UDP]": custom STUN server (host:port); on macOS checkbox "Use system SOCKS5 proxy" (via proxy or direct). "?" button by the input opens the list of public STUN servers [always-online-stun](https://github.com/pradt2/always-online-stun?tab=readme-ov-file#always-online-stun-servers). STUN button full width with gear on the right; in dialog input field full width with "?" on the right.

**Diagnostics & Help**

- **Diagnostics tab** — "Open logs" split into two buttons: "Open log window" (window icon) and "Open logs folder" (folder icon). Clicking "Open log window" when the log window is already open focuses it instead of opening a duplicate.
- **Help tab** — "Open Logs Folder" button removed (logs folder is available from Diagnostics).

**Fixes**

- **Sources disappearing** — Fixed after reopening wizard, changing prefixes, and saving. ParseAndPreview no longer applies the URL field to ParserConfig; preview and save use only ParserConfig.Proxies as the source of truth. TriggerParseForPreview no longer requires the URL field to be non-empty.
- **Comments/docs** — Clarified: ParserConfig is the source of truth for sources; SourceURLs is input for the Add button only.
