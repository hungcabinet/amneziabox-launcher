## ✨ Новые функции

### Wizard - Управление состояниями
- **Система сохранения и загрузки состояний визарда**: Реализована полная система работы с состояниями конфигурации через `state.json`
  - Сохранение текущего состояния визарда
  - Загрузка сохраненных состояний
  - Работа с несколькими конфигурациями
  - Восстановление работы с того места, где остановились
  - Автоматическое создание директории `bin/wizard_states/`
  - Валидация версии формата и размера файла
  - 📁 [SPECS/002-F-C-WIZARD_STATE/](SPECS/002-F-C-WIZARD_STATE/)

### Wizard - Get free VPN
- **Кнопка "Get free VPN!"**: Добавлена кнопка в Sources tab для автоматического получения бесплатных VPN конфигураций
  - Автоматическое скачивание файла `get_free.json` с GitHub при необходимости
  - Загрузка данных из локального файла или автоматическое скачивание
  - Извлечение URLs из `ParserConfig.proxies` и автоматическая вставка в поле "Subscription URL or Direct Links"
  - Вставка полного `ParserConfig` из файла в поле конфигурации
  - Гарантия, что ParserConfig из файла не перезаписывается автоматически сгенерированным
  - Полная обработка ошибок на всех этапах
  - Автоматическое восстановление при повреждении файла
  - Валидация данных перед использованием
  - Благодарность [igareck](https://github.com/igareck) за проект [vpn-configs-for-russia](https://github.com/igareck/vpn-configs-for-russia)
  - 📄 [SPECS/001-F-C-FEATURES_2025/2026-02-10-get-free-vpn-button.md](SPECS/001-F-C-FEATURES_2025/2026-02-10-get-free-vpn-button.md)

### Wizard - Автоматический парсинг
- **Автоматический парсинг при переключении на Rules tab**: При переключении на вкладку Rules автоматически запускается парсинг ParserConfig для гарантии актуальности списка outbounds

### Проверка обновлений
- **Автоматическая проверка обновлений при запуске**: Приложение автоматически проверяет наличие новых версий при запуске
  - Проверка выполняется асинхронно в фоне, не блокируя запуск приложения
  - Результат проверки сохраняется в кеш
  - При наличии новой версии показывается попап с информацией об обновлении
  - Попап отображается при первом открытии главного окна (через `OnWindowShown` callback)
  - Попап содержит текущую и новую версию, а также ссылку на скачивание с GitHub
  - Попап показывается только один раз за сессию
  - UI-код отделен от бизнес-логики (попап в `ui/core_dashboard_tab.go`)
  - Используется `components.NewCustom` для отображения диалога
  - 📄 [SPECS/001-F-C-FEATURES_2025/UPDATE_CHECK_POPUP.md](SPECS/001-F-C-FEATURES_2025/UPDATE_CHECK_POPUP.md)

### Новые типы правил
- **Тип правил "Processes"**: Добавлен новый тип правил для работы с процессами
  - Селектор активных процессов в отдельном попапе (Refresh / +Add)
  - Поддержка множественного выбора
  - Дедупликация и сортировка по имени
  - Поле отображается на ~4 строки с возможностью удаления по строкам
  - Правило сохраняется под ключом `process_name`

- **Тип правил "Custom JSON"**: Добавлен тип правила для произвольного JSON
  - Многострочное поле для ввода/правки JSON
  - Валидация JSON перед сохранением
  - JSON должен быть объектом (map)
  - Объект используется как полное тело `Raw` правила
  - Автоматическое определение Custom правил при редактировании

## 🔧 Улучшения

### UI - Wizard
- **Wizard single-instance**: Окно конфигурационного визарда теперь может быть открыто только в одном экземпляре — повторный вызов фокусирует существующее окно вместо создания нового
- **Click-redirect overlay**: Добавлен невидимый оверлей поверх основного окна, который при клике переводит фокус на открытый визард
- **Open from tray**: При выборе "Open" в трее главное окно всегда восстанавливается; если визард открыт — он отображается поверх главного окна и получает фокус
- **Автоматическая проверка URL**: Автоматическая проверка URL при изменении ввода с задержкой 2 секунды; кнопка "Check" удалена для улучшения UX
- **Chat GPT button**: Добавлена кнопка "Chat GPT" в Sources tab, которая создает URL-encoded промпт с ParserConfig и открывает ChatGPT (также копирует промпт в буфер обмена)

### Servers (Clash API)
- **Per-selector last proxy**: Приложение теперь запоминает последний выбранный прокси для каждой группы селектора (улучшение по сравнению с глобальным сохранением в 0.7.0)
- **Selector → Active Outbound (live)**: Добавлена иконка информации рядом с выпадающим списком селектора, которая запрашивает Clash API и показывает текущий активный outbound для каждого селектора
- **UX**: Вкладка Servers теперь отображает "Last used proxy" для текущей выбранной группы

### macOS
- **Hide app from Dock**: Добавлена функция скрытия приложения из Dock. Когда скрыто, приложение продолжает работать в трее; открытие приложения восстанавливает иконку в Dock
  - Реализация использует darwin-специфичный CGO helper с безопасными stubs для других платформ
  - https://github.com/Leadaxe/singbox-launcher/pull/23 (спасибо https://github.com/MustDie-green)

### Файловые операции
- **Централизация бэкапа файлов**: Логика создания бэкапов вынесена в `core/services/file_service.go`
  - Единый лимит на количество старых файлов (1 старый файл для логов и общих бэкапов)
  - Автоматическая ротация старых бэкапов
  - Упрощённая логика работы с файлами

## 🐛 Исправления ошибок

### Wizard
- **Исправлено удаление подписок**: Config Wizard теперь корректно удаляет удаленные subscription URLs и direct links при редактировании ввода в визарде. Ранее удаленные строки могли оставаться в сгенерированном `ParserConfig`; теперь исправлено для полных и частичных удалений
- **Исправлена генерация невалидного JSONC**: Исправлен баг, когда при отсутствии сгенерированных outbounds статические outbounds начинали с запятой, что приводило к невалидному JSONC (`[, ...]`)
- **Исправлено обновление UI вкладки Rules после LoadState**: Исправлена проблема, когда чекбоксы и селекторы outbound не обновлялись после загрузки состояния из `get_free.json` или через кнопку "Read". Реализовано через Dependency Injection: функция создания вкладки Rules хранится в презентере и используется для пересоздания вкладки после `LoadState()` через `SyncModelToGUI()`

### Валидация и UX
- **Валидация правил**: Кнопка **Add/Save** активна только при заполненном имени правила и наличии данных для выбранного типа (для `Processes` — хотя бы один выбранный процесс)

## ✨ New Features

### Wizard - State Management
- **Wizard state save/load system**: Full state management system for configuration via `state.json`
  - Save current wizard state
  - Load saved states
  - Work with multiple configurations
  - Resume work from where you left off
  - Automatic creation of `bin/wizard_states/` directory
  - Format version and file size validation
  - 📁 [SPECS/002-F-C-WIZARD_STATE/](SPECS/002-F-C-WIZARD_STATE/)

### Wizard - Get free VPN
- **"Get free VPN!" button**: Added button in Sources tab for automatic free VPN configuration retrieval
  - Automatic download of `get_free.json` from GitHub when needed
  - Load data from local file or automatic download
  - Extract URLs from `ParserConfig.proxies` and automatically insert into "Subscription URL or Direct Links" field
  - Insert full `ParserConfig` from file into configuration field
  - Guarantee that ParserConfig from file is not overwritten by auto-generated one
  - Full error handling at all stages
  - Automatic recovery for corrupted files
  - Data validation before use
  - Thanks to [igareck](https://github.com/igareck) for the [vpn-configs-for-russia](https://github.com/igareck/vpn-configs-for-russia) project
  - 📄 [SPECS/001-F-C-FEATURES_2025/2026-02-10-get-free-vpn-button.md](SPECS/001-F-C-FEATURES_2025/2026-02-10-get-free-vpn-button.md)

### Wizard - Auto-parse
- **Auto-parse on Rules tab switch**: When switching to Rules tab, ParserConfig parsing is automatically triggered to ensure outbounds list is up-to-date

### Update Check
- **Automatic update check on startup**: Application automatically checks for new versions on startup
  - Check runs asynchronously in background, not blocking application startup
  - Check result is cached
  - If new version is available, a popup with update information is shown
  - Popup is displayed on first main window display (via `OnWindowShown` callback)
  - Popup contains current and new version, as well as download link to GitHub
  - Popup is shown only once per session
  - UI code separated from business logic (popup in `ui/core_dashboard_tab.go`)
  - Uses `components.NewCustom` for dialog display
  - 📄 [SPECS/001-F-C-FEATURES_2025/UPDATE_CHECK_POPUP.md](SPECS/001-F-C-FEATURES_2025/UPDATE_CHECK_POPUP.md)

### New Rule Types
- **"Processes" rule type**: Added new rule type for working with processes
  - Active process selector in separate popup (Refresh / +Add)
  - Multiple selection support
  - Deduplication and sorting by name
  - Field displays ~4 lines with ability to remove by lines
  - Rule saved under `process_name` key

- **"Custom JSON" rule type**: Added rule type for arbitrary JSON
  - Multi-line field for JSON input/editing
  - JSON validation before saving
  - JSON must be an object (map)
  - Object used as full `Raw` rule body
  - Automatic Custom rule detection when editing

## 🔧 Improvements

### UI - Wizard
- **Wizard single-instance**: Configuration wizard window can now only be opened in one instance — repeated call focuses existing window instead of creating new one
- **Click-redirect overlay**: Added invisible overlay over main window that redirects clicks to open wizard
- **Open from tray**: When selecting "Open" in tray, main window always restores; if wizard is open — it displays above main window and receives focus
- **Automatic URL checking**: Automatic URL checking on input change with 2 second debounce; "Check" button removed to improve UX
- **Chat GPT button**: Added "Chat GPT" button in Sources tab that builds URL-encoded prompt with ParserConfig and opens ChatGPT (also copies prompt to clipboard)

### Servers (Clash API)
- **Per-selector last proxy**: App now remembers last selected proxy per selector group (improvement over global saving in 0.7.0)
- **Selector → Active Outbound (live)**: Added info icon next to selector dropdown that queries Clash API and shows current active outbound for each selector
- **UX**: Servers tab now displays "Last used proxy" for currently selected group

### macOS
- **Hide app from Dock**: Added feature to hide app from Dock. When hidden, app continues running in tray; opening app restores Dock icon
  - Implementation uses darwin-specific CGO helper with safe stubs for other platforms
  - https://github.com/Leadaxe/singbox-launcher/pull/23 (thanks https://github.com/MustDie-green)

### File Operations
- **Centralized file backup**: File backup logic extracted to `core/services/file_service.go`
  - Unified limit on number of old files (1 old file for logs and general backups)
  - Automatic rotation of old backups
  - Simplified file handling logic

- **Unified logging system**: All `log.*` calls replaced with `debuglog.*` semantic functions
  - `ErrorLog()` - for errors
  - `WarnLog()` - for warnings
  - `InfoLog()` - for information
  - `DebugLog()` - for debugging
  - Simplified API (fewer parameters)
  - Automatic level prefix addition
  - Timing utility (`StartTiming()`)
  - Log level control via `SINGBOX_DEBUG` environment variable
  - 30 files changed, 541 lines added, 402 lines removed

### Singleton Controller
- **Implemented `GetController()` as singleton**: Removed `*AppController` parameter from all `core.*` functions
  - Simplified API of all `core.*` functions
  - Reduced code duplication
  - Centralized controller access
  - Thread-safety via `sync.Once`
  - Removed `controller` field from `WizardPresenter`
  - 📄 [SPECS/001-F-C-FEATURES_2025/SINGLETON_CONTROLLER.md](SPECS/001-F-C-FEATURES_2025/SINGLETON_CONTROLLER.md)

- **`internal/process` package**: Process enumeration logic extracted to separate package
  - Wrapper over `go-ps`
  - Removed duplicate implementations

### Unified config_template.json
- **Unified configuration template**: Completely redesigned wizard template system
  - Removed `config_template_macos.json` — now uses single `config_template.json` for all platforms
  - New JSON structure with 4 sections: `parser_config`, `config`, `selectable_rules`, `params`
  - Removed fragile comment-based directives (`@ParserConfig`, `@SelectableRule`, `@PARSER_OUTBOUNDS_BLOCK`)
  - Platform-specific settings via `params` section (automatically applied based on `runtime.GOOS`)
  - Rules with platform filtering via `platforms` field in `selectable_rules`
  - `rule_set` definitions now inside each rule (only loaded when rule is enabled)
  - Simplified `state.json` (version 2): `selectable_rule_states` store only `label`, `enabled`, `selected_outbound`
  - Automatic migration from old `state.json` format
  - Removed all regex-based comment parsing
  - Template is now valid JSON, parsed with standard `json.Unmarshal`
  - Improved template readability and maintainability
  - All documentation updated to reflect new structure
  - Removed `cloneRule()` and `normalizeProcessNames()` functions (no longer needed thanks to `platforms`)
  - 📁 [SPECS/003-F-C-UNIFIED_CONFIG_TEMPLATE/](SPECS/003-F-C-UNIFIED_CONFIG_TEMPLATE/)

### controller.go Refactoring
- **Separation of concerns**: Logic from `controller.go` extracted to separate files

  - `core/auto_update.go` — auto-update configuration logic
  - Consolidated error display methods in `core/error_handler.go`

## 🐛 Bug Fixes

### Wizard
- **Fixed subscription removal**: Config Wizard now properly removes deleted subscription URLs and direct links when editing input in wizard. Previously removed lines could remain in generated `ParserConfig`; now fixed to respect full and partial deletions
- **Fixed invalid JSONC generation**: Fixed bug where static outbounds started with comma when no generated outbounds were present, causing invalid JSONC (`[, ...]`)
- **Fixed Rules tab UI update after LoadState**: Fixed issue where checkboxes and outbound selectors were not updated after loading state from `get_free.json` or via "Read" button. Implemented via Dependency Injection: Rules tab creation function is stored in presenter and used to recreate the tab after `LoadState()` through `SyncModelToGUI()`

### Validation and UX
- **Rule validation**: **Add/Save** button is only active when rule name is filled and data exists for selected type (for `Processes` — at least one selected process)

- Added Windows-friendly cache paths for Go build and modules
