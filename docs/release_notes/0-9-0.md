# What's New from 0.8.6

## Version 0.9.0 Release Notes

---

## EN

### Highlights

- **Debug API (localhost-only):** optional HTTP server on `127.0.0.1:9269`. **Off by default.** Read endpoints `/ping`, `/version`, `/state`, `/proxies`; action endpoints `/action/{start,stop,update-subs,ping-all}` (POST-only). Bearer-token auth (32-byte `crypto/rand`, constant-time compare). Toggle + "Copy token" button on the Diagnostics tab, with a confirmation toast on copy. Listener strictly `127.0.0.1` — remote access via `adb forward` / ssh-tunnel only. Primary use-case: a backing service for **MCP wrappers** so AI agents (Claude / Cursor / etc.) can read and drive the launcher. Ported from LxBox spec 031, trimmed for desktop (no rules/subs CRUD, no `/config` dump, no `/logs` tail). See [**SPEC 038**](../../SPECS/038-F-C-DEBUG_API/SPEC.md).

- **New ⚙️ Settings tab** between Servers and Diagnostics. Consolidates launcher-wide preferences that were scattered across Core Dashboard and Help: **language selector** + "Download locales" (moved from Help), **Auto-update subscriptions** toggle, **Auto-ping on connect** toggle. Fixes a latent data-loss bug in the old language handler — previously `SaveSettings({Lang: code})` wiped every other settings field; now uses load-mutate-save. See [**SPEC 039**](../../SPECS/039-F-C-SETTINGS_TAB_PREFERENCES/SPEC.md).

- **Subscription source toggle** in the Wizard → Sources list. Per-row on/off checkbox; disabled sources are skipped by the parser entirely (no fetch, no parse) but stay in the file — URL, skip rules, tag prefix preserved for easy re-enable. `ProxySource.Disabled` with `omitempty` keeps legacy configs compatible. See [**SPEC 037**](../../SPECS/037-F-C-SUBSCRIPTION_SOURCE_TOGGLE/SPEC.md).

- **Auto-ping after VPN connect** (default ON). 5 seconds after sing-box enters the running state, the Servers tab auto-runs Test on all proxies, so latency numbers are fresh when you switch tabs. Timer is cancelled if you stop sing-box inside the window. Persisted as `auto_ping_after_connect_disabled` in `settings.json`. Part of [SPEC 039](../../SPECS/039-F-C-SETTINGS_TAB_PREFERENCES/SPEC.md).

- **Subscription auto-update global toggle** (default ON). Off skips all scheduled refreshes; manual Update always works. Re-enabling from the UI also resets the consecutive-failure counter so the loop does not auto-disable itself again. Persisted as `subscription_auto_update_disabled`.

- **URLTest parameters as template vars** with preset dropdowns. `auto-proxy-out.url / interval / tolerance` hoisted to `@urltest_url / @urltest_interval / @urltest_tolerance`; values are editable in Wizard → Settings instead of requiring manual JSON edits. Sensible presets and units labelled. See [**SPEC 040**](../../SPECS/040-F-C-WIZARD_TEMPLATE_OPTION_TITLES/SPEC.md).

- **Template `vars[].options` accepts `{title, value}` form.** User-visible dropdown labels can now differ from the substituted value (e.g. `"5m (default)"` displayed, `"5m"` goes into the config). Legacy `["a", "b"]` string-list form still works (title == value). Mixed arrays supported. See [SPEC 040](../../SPECS/040-F-C-WIZARD_TEMPLATE_OPTION_TITLES/SPEC.md).

- **Keyboard shortcuts.** `Cmd/Ctrl+R` reconnect sing-box, `Cmd/Ctrl+U` update subscriptions, `Cmd/Ctrl+P` ping-all. Modifier resolves automatically per platform (Super on macOS, Ctrl on Win/Linux) via `fyne.KeyModifierShortcutDefault`. Focused text entries (e.g. inside Wizard) still swallow keystrokes as usual. See [**SPEC 042**](../../SPECS/042-F-C-KEYBOARD_SHORTCUTS/SPEC.md).

- **Power-resume hooks on Windows + Linux** (partial closure of [SPEC 011](../../SPECS/011-B-C-launcher-freeze-after-sleep/SPEC.md)). After sleep / hibernate, the launcher resets the Clash API HTTP transport, refreshes the proxies list, and — if the tunnel is up and auto-ping is enabled — re-pings nodes so latency numbers are not stuck pre-sleep. Linux gained native support via **systemd-logind `PrepareForSleep`** over the system DBus (reuses the existing `github.com/godbus/dbus/v5` indirect dep — no new dependencies). On Linux systems without logind (minimal distros, WSL without logind) the listener silently no-ops. **macOS IOKit hook remains TODO** for a dedicated branch (cgo complexity).

- **Atomic writes** for `config.json` (scheduled parser + wizard save) and `settings.json`. Stage to `.tmp` / `.swap` then atomic `os.Rename` — a crash, kill -9, or power loss mid-write can no longer truncate your live config. The wizard also keeps its existing `.bak` backup for manual recovery. See [**SPEC 041**](../../SPECS/041-F-C-ATOMIC_FILE_WRITES/SPEC.md).

### Added — Resilience & observability

- **100 MB download cap** on sing-box core downloads. Pre-flight Content-Length check + mid-stream cumulative counter. Protects against a compromised or misconfigured mirror filling the user's drive before any error fires. Legitimate sing-box release archives are < 20 MB.

- **HTTP status humanization** in subscription-fetch errors: `401 → "unauthorized — subscription token may have expired"`, `403 → "forbidden — provider blocked this request"`, `404 / 410 / 429 / 5xx` each with a short hint. Helps triage without googling.

- **"(subs: Xh ago)" freshness hint** next to the `config.json` row on Core Dashboard — how long since the last successful subscription update. Session-scoped (resets on launcher restart).

- **Dirty-config marker (`*`)** on the Update button when the Wizard has saved changes that the parser has not yet applied. See [**SPEC 043**](../../SPECS/043-F-C-DIRTY_CONFIG_MARKER/SPEC.md). The current implementation is the minimal version — it triggers on any Wizard save; a cleaner design that distinguishes "sources changed (need parser)" from "template changed (need restart)" is tracked in the spec.

### Added — Security hygiene

- **Clash API token redaction** in debug logs. `LoadClashAPIConfig` used to log the full bearer token on every start; now logs `token=ab***89` (first 2 + masked + last 2 chars; fully masked for tokens ≤ 6 chars). Token remains visible in the Settings UI via Copy button. Users pasting debug logs into GitHub issues / Telegram support no longer leak their Clash secret.

### Fixed

- **CI red since 2026-04-16** — `internal/locale.TestAllKeysPresent` was failing because commit `2f77bf7` added 4 new English context-menu keys for "Copy jump server link" without the Russian counterparts. Parity restored.
- **STUN settings dialog** clipping on Windows ([issue #54](https://github.com/Leadaxe/singbox-launcher/issues/54)) — auto-sizing collapsed the URL entry below placeholder width. Dialog now explicitly sized to ≥ 520 px. Dedup'd the darwin / non-darwin branches along the way.

### Changed — Template defaults

- **Russian & Cyrillic TLDs expanded** in the `ru-domains` rule-set. Was `.ru / .xn--p1ai / .su`. Added `.рус (xn--p1acf) / .москва (xn--80adxhks) / moscow / tatar / .дети / .сайт / .орг / .ком`. Users picking up the new template via the Download Template button automatically get the expanded list; existing custom templates are unaffected.

### Technical / Internal

- **`StateService` extensions:** `AutoPingAfterConnect` (bool, default true) with mutex + Is/Set; `TemplateDirty` (bool) for dirty-config marker; `LastUpdateError / LastUpdateErrorAt / LastUpdateSucceededAt` for future failure-pill redesign.
- **`ProxySource.Disabled bool json:"disabled,omitempty"`** — new field, omit-default preserves legacy configs.
- **`OutboundGenerationResult` extensions:** `TotalSources / SucceededSources / FailedSources` counters (only `TotalSources` surfaced in user-visible text for now — per-source toast redesign is post-release).
- **`TemplateVar` custom `UnmarshalJSON`** — supports three JSON forms for `options`. `OptionTitles []string` parallel to `Options` when titles present.
- **New package `core/debugapi/`** — isolated HTTP surface with narrow `ControllerFacade` interface (no import cycle on `core/`). Adapter `debugAPIFacade` in `core/debugapi_wiring.go`.
- **`internal/platform/power_linux.go`** — new file, build tag `linux`. `power_stub.go` build tag tightened to `!windows && !linux`.
- **Closed / extended SPECs:** [011](../../SPECS/011-B-C-launcher-freeze-after-sleep/SPEC.md) (extension), [037](../../SPECS/037-F-C-SUBSCRIPTION_SOURCE_TOGGLE/SPEC.md), [038](../../SPECS/038-F-C-DEBUG_API/SPEC.md), [039](../../SPECS/039-F-C-SETTINGS_TAB_PREFERENCES/SPEC.md), [040](../../SPECS/040-F-C-WIZARD_TEMPLATE_OPTION_TITLES/SPEC.md), [041](../../SPECS/041-F-C-ATOMIC_FILE_WRITES/SPEC.md), [042](../../SPECS/042-F-C-KEYBOARD_SHORTCUTS/SPEC.md), [043](../../SPECS/043-F-C-DIRTY_CONFIG_MARKER/SPEC.md).

### Migration notes

- `bin/settings.json` gains new optional fields: `subscription_auto_update_disabled`, `auto_ping_after_connect_disabled`, `debug_api_enabled`, `debug_api_token`, `debug_api_port`. Missing fields are treated as zero (enabled / empty / 9269), so existing installs behave as before until you toggle.
- `wizard_template.json` — `auto-proxy-out` options now reference `@urltest_url / @urltest_interval / @urltest_tolerance`. If you maintain a custom template, either merge the three new `vars` definitions from the shipped template or keep your literal values (both forms work).
- Language selector moved from **Help** to the new **Settings** tab. Existing users who look for it in Help will not find it — see [SPEC 039](../../SPECS/039-F-C-SETTINGS_TAB_PREFERENCES/SPEC.md).

### Known issues / deferred for redesign

These items are **not** in 0.9.0; their spec sections document the redesign plan:

- Right-click context menu on the Update button — `SecondaryTapWrap` does not route secondary taps into `widget.Button`; needs a custom `RightClickableButton`.
- Per-source parser success/failure summary in the toast — counters exist in `OutboundGenerationResult`, but the user-visible toast still reads "Config updated successfully" regardless.
- Last-auto-update failure pill on Core Dashboard — wired but not visible in practice; layout / flag-path diagnosis pending.
- Ping-all button cancel — current design fully disables the button during a run; a click-to-cancel redesign (button label swap to "Cancel" / "Cancelling…" with `pingAllCancelled` flag) is tracked.
- Wizard URLTest-vars widget rendering — `{title, value}` options in an editable `SelectEntry` is semantically broken (user can type arbitrary text into a labeled preset). Rule for the next release: `enum` → `widget.NewSelect` (titles allowed); `text` + options → `SelectEntry` (plain strings only); `text` + `{title, value}` → validator error.
- macOS IOKit power-resume hook — follow-up to [SPEC 011](../../SPECS/011-B-C-launcher-freeze-after-sleep/SPEC.md).
- Tooltip hints on buttons bound to keyboard shortcuts — discoverable reminder of `Cmd+R / Cmd+U / Cmd+P`.

## RU

### Основное

- **Debug API (локальный HTTP):** опциональный сервер на `127.0.0.1:9269`. **По умолчанию выключен.** Read-эндпоинты `/ping`, `/version`, `/state`, `/proxies`; action-эндпоинты `/action/{start,stop,update-subs,ping-all}` (только POST). Bearer-токен (32 байта `crypto/rand`, constant-time compare). Чекбокс + кнопка «Копировать токен» на вкладке Диагностика с подтверждением-тостом. Listener строго на `127.0.0.1` — удалённый доступ только через `adb forward` / ssh-tunnel. Основной use-case: backing-service для **MCP-обёрток**, чтобы AI-агенты (Claude / Cursor / …) могли читать и дёргать лаунчер. Портировано из LxBox spec 031 в урезанном виде (без CRUD правил/подписок, без `/config`, без `/logs`). См. [**SPEC 038**](../../SPECS/038-F-C-DEBUG_API/SPEC.md).

- **Новая вкладка ⚙️ «Настройки»** между Servers и Diagnostics. Собирает launcher-wide preferences, ранее разбросанные по Core Dashboard и Help: **селектор языка** + «Скачать переводы» (перенесены из Help), **автообновление подписок**, **автопинг после подключения**. Попутно починен data-loss баг старого language-handler'а — раньше `SaveSettings({Lang: code})` затирал все остальные поля; теперь load-mutate-save. См. [**SPEC 039**](../../SPECS/039-F-C-SETTINGS_TAB_PREFERENCES/SPEC.md).

- **Переключатель подписки** в Wizard → Sources. Чекбокс в каждой строке; отключённые источники полностью пропускаются парсером (без fetch, без parse), но остаются в файле — URL / skip / tag prefix сохраняются для быстрого повторного включения. `ProxySource.Disabled` с `omitempty` — legacy-конфиги совместимы. См. [**SPEC 037**](../../SPECS/037-F-C-SUBSCRIPTION_SOURCE_TOGGLE/SPEC.md).

- **Автопинг после подключения VPN** (по умолчанию ВКЛ). Через 5 с после перехода sing-box в running вкладка Servers авто-запускает Test для всех прокси — когда пользователь переключится туда, задержки уже свежие. Таймер отменяется, если пользователь нажал Stop внутри окна. Хранится как `auto_ping_after_connect_disabled` в `settings.json`. Часть [SPEC 039](../../SPECS/039-F-C-SETTINGS_TAB_PREFERENCES/SPEC.md).

- **Глобальный выключатель автообновления подписок** (по умолчанию ВКЛ). OFF — плановые обновления пропускаются; ручной Update всегда работает. Включение через UI сбрасывает счётчик последовательных ошибок. Хранится как `subscription_auto_update_disabled`.

- **Параметры URLTest как шаблонные vars** с preset-дропдаунами. `auto-proxy-out.url / interval / tolerance` вынесены в `@urltest_url / @urltest_interval / @urltest_tolerance`; редактируются в Wizard → Settings вместо ручной правки JSON. Разумные пресеты с подписями (например, «5m (default)», «100 ms (default)»). См. [**SPEC 040**](../../SPECS/040-F-C-WIZARD_TEMPLATE_OPTION_TITLES/SPEC.md).

- **`vars[].options` поддерживает форму `{title, value}`.** Подписи в дропдаунах теперь могут отличаться от подставляемого значения (например, на экране «5m (default)», в конфиг уйдёт `"5m"`). Старая форма `["a", "b"]` продолжает работать (title == value). Смешанные массивы допустимы. См. [SPEC 040](../../SPECS/040-F-C-WIZARD_TEMPLATE_OPTION_TITLES/SPEC.md).

- **Горячие клавиши.** `Cmd/Ctrl+R` — reconnect sing-box, `Cmd/Ctrl+U` — update subscriptions, `Cmd/Ctrl+P` — ping-all. Modifier автоматически маппится на платформу (Super на macOS, Ctrl на Win/Linux) через `fyne.KeyModifierShortcutDefault`. Фокусированный Entry (например, внутри визарда) как обычно проглатывает keystroke. См. [**SPEC 042**](../../SPECS/042-F-C-KEYBOARD_SHORTCUTS/SPEC.md).

- **Обработка wake-from-sleep на Windows + Linux** (частичное закрытие [SPEC 011](../../SPECS/011-B-C-launcher-freeze-after-sleep/SPEC.md)). После сна / гибернации лаунчер сбрасывает HTTP-транспорт Clash API, перечитывает список прокси и — если VPN работает и автопинг включён — re-пингует ноды, чтобы задержки не застывали до сна. Linux получил нативную поддержку через **systemd-logind `PrepareForSleep`** по system DBus (используется уже существующий `github.com/godbus/dbus/v5` indirect-dep, новых зависимостей нет). На Linux без logind (минимальные дистрибутивы, WSL без logind) listener молча no-op'ит. **macOS IOKit-хук — TODO в отдельной ветке** (cgo complexity).

- **Атомарные записи** `config.json` (плановый парсер + wizard save) и `settings.json`. Сначала `.tmp` / `.swap`, затем атомарный `os.Rename` — падение, kill -9 или обесточивание посреди записи не могут обнулить живой конфиг. Визард дополнительно сохраняет свой `.bak` для ручного восстановления. См. [**SPEC 041**](../../SPECS/041-F-C-ATOMIC_FILE_WRITES/SPEC.md).

### Добавлено — устойчивость и наблюдаемость

- **Лимит 100 МБ** на загрузку sing-box core. Проверка Content-Length до скачивания + кумулятивный счётчик во время. Защищает от скомпрометированного или неправильно настроенного зеркала, которое может залить диск пользователя до момента первой ошибки. Легитимные архивы релиза sing-box < 20 МБ.

- **Расшифровка HTTP-статусов** в ошибках subscription-fetch: `401 → "unauthorized — subscription token may have expired"`, `403 → "forbidden — provider blocked this request"`, `404 / 410 / 429 / 5xx` — каждый с короткой подсказкой. Помогает триажить без Google.

- **Подсказка «(подписки: X ч назад)»** рядом со строкой `config.json` на Core Dashboard — сколько прошло с последнего успешного обновления. В рамках сессии (сбрасывается на перезапуск лаунчера).

- **Маркер `*`** на кнопке Update, когда визард сохранил изменения, а парсер ещё не прокатал. См. [**SPEC 043**](../../SPECS/043-F-C-DIRTY_CONFIG_MARKER/SPEC.md). Текущая реализация — минимальная; она загорается на любом wizard Save. Более корректный дизайн, различающий «менялись sources (нужен парсер)» и «менялся template (нужен restart)», трекается в спеке.

### Добавлено — безопасность

- **Редакция токена Clash API** в debug-логах. Раньше `LoadClashAPIConfig` писал полный bearer-токен в лог при каждом старте; теперь пишется `token=ab***89` (первые 2 + маска + последние 2; полностью маскируется при длине ≤ 6). Токен по-прежнему виден в UI через кнопку Copy. Пользователи, публикующие debug-логи в GitHub issues / Telegram-support, больше не утекают свой Clash-secret.

- **STUN settings dialog** обрезался на Windows ([issue #54](https://github.com/Leadaxe/singbox-launcher/issues/54)) — auto-size сжимал URL-entry под placeholder-ширину. Диалог теперь явно ≥ 520 px. Попутно деудублены darwin / non-darwin ветки.

Добавлены: `.рус (xn--p1acf) / .москва (xn--80adxhks) / moscow / tatar / .дети / .сайт / .орг / .ком`. Пользователи, скачивающие новый template через кнопку «Download Template», получают расширенный список; кастомные шаблоны не затрагиваются.

### Миграция

- В `bin/settings.json` добавлены новые опциональные поля: `subscription_auto_update_disabled`, `auto_ping_after_connect_disabled`, `debug_api_enabled`, `debug_api_token`, `debug_api_port`. Отсутствие трактуется как zero (enabled / пусто / 9269), поэтому существующие установки ведут себя как раньше до первого toggle.

### Known issues / отложено на редизайн

Не входит в 0.9.0; в соответствующих спеках описан план редизайна:

- Контекстное меню по правому клику на Update — `SecondaryTapWrap` не маршрутизирует secondary-тапы в `widget.Button`; нужен кастомный `RightClickableButton`.
- Per-source сводка успехов/неудач в тосте парсера — счётчики доходят до `OutboundGenerationResult`, но user-visible тост по-прежнему пишет «Config updated successfully» независимо от реального результата.
- Плитка «Last auto-update failed» на Core Dashboard — wired, но в UI не видна; диагностика layout / flag-path отложена.
- Отмена ping-all — текущий дизайн полностью disables кнопку во время прогона; click-to-cancel редизайн (swap лейбла на «Cancel» / «Cancelling…» + флаг `pingAllCancelled`) трекается.
- Рендеринг виджетов URLTest-vars в визарде — `{title, value}` в editable `SelectEntry` семантически некорректен (пользователь может напечатать произвольный текст в поле с пресет-подписью). Правило для следующего релиза: `enum` → `widget.NewSelect` (titles допустимы); `text` + options → `SelectEntry` (только plain strings); `text` + `{title, value}` → ошибка валидатора.
- macOS IOKit power-resume хук — follow-up к [SPEC 011](../../SPECS/011-B-C-launcher-freeze-after-sleep/SPEC.md).
- Tooltip-подсказки на кнопках с keyboard-шортками — discoverable напоминание о `Cmd+R / Cmd+U / Cmd+P`.
