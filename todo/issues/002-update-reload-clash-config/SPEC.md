# Спецификация: сохранение в визарде — только сохранение файлов и Update

Поведение при сохранении в визарде должно быть строго определённым: сохранение файлов и вызов **Update**; без какого-либо дополнительного поведения (в частности, без перезапуска). После успешного Update — перечитывать настройки Clash API из обновлённого `config.json`, чтобы лаунчер подхватывал новый конфиг без рестарта sing-box.

## Проблема

1. **Сохранение в визарде:** при сохранении в визарде допустимо только: запись файлов и вызов Update. Не должно быть дополнительного поведения (перезапуск sing-box, перезапуск приложения и т.п.).
2. **После Update:** по нажатию **Update** выполняется только `RunParserProcess()`: подтягиваются подписки, пересобираются outbounds, записывается `config.json`. Состояние лаунчера (вкладка Servers, Clash API — baseURL, token, группы селекторов) при этом не обновляется из нового файла. Чтобы вкладка Servers и селекторы соответствовали обновлённому конфигу, пользователю приходится перезапускать sing-box или приложение.

## Текущее поведение (как сейчас)

При сохранении в визарде (`ui/wizard/presentation/presenter_save.go`):

1. **Сохранение файлов:** сборка конфига из модели → запись `config.json` → валидация sing-box → запись `state.json`.
2. **Обновление статуса в Core:** вызов `UpdateConfigStatusFunc()`.
3. **Дополнительное поведение — перезапуск sing-box:** в `saveStateAndShowSuccessDialog()` (стр. 254–281) при запущенном sing-box выполняется `StopSingBoxProcess()`, затем в горутине после паузы 500 ms — ожидание остановки (до 2 с) и `StartSingBoxProcess(true)`.
4. **Update:** в `completeSaveOperation()` в фоне вызывается `go core.RunParserProcess()` — тот же поток, что и кнопка Update на главной вкладке.

Итого: сейчас делаются сохранение файлов и вызов Update, но дополнительно выполняется перезапуск sing-box при его запущенном состоянии. По контракту перезапуск не должен входить в сохранение в визарде.

## Цель

1. **Визард:** при сохранении в визарде — только сохранение файлов и вызов Update; никакого дополнительного поведения (в т.ч. перезапуска).
2. **После успешного Update:** перечитывать конфигурацию Clash API из обновлённого `config.json` (через `APIService.ReloadClashAPIConfig()`), чтобы лаунчер подхватывал актуальные `experimental.clash_api` и список селекторов без рестарта sing-box или приложения.

## Требования

1. **Сохранение в визарде:** при сохранении в визарде выполнять только сохранение файлов и вызов Update (тот же поток, что и кнопка Update). Не выполнять перезапуск sing-box, перезапуск приложения или иное дополнительное поведение.
2. **После успешного RunParserProcess (Update):** вызывать `ReloadClashAPIConfig()` (на контроллере: `ac.APIService.ReloadClashAPIConfig()`), чтобы перечитать из `config.json` настройки Clash API и при необходимости обновить `SelectedClashGroup` по умолчанию.
3. Вызов `ReloadClashAPIConfig` — только при успешном завершении обновления конфига (в существующем пути успеха RunParserProcess / UpdateConfigFromSubscriptions).
4. Ошибку `ReloadClashAPIConfig` не показывать пользователю как блокирующую (конфиг уже сохранён); при необходимости логировать и продолжать.

## Критерии приёмки

- При сохранении в визарде выполняются только сохранение файлов и вызов Update; перезапуск и прочее дополнительное поведение отсутствуют.
- После нажатия Update (и после успешного Update из визарда) и успешного обновления конфига вызывается `ReloadClashAPIConfig()`.
- Вкладка Servers и селекторы после Update отражают данные из обновлённого `config.json` без рестарта sing-box/лаунчера.
- Существующее поведение при ошибке Update не меняется.
- Рестарт sing-box после Update не требуется и не добавляется.

## Замечания

- При сохранении в визарде контракт: только запись файлов и вызов Update; перезапуск не входит в контракт и не должен выполняться.
- sing-box сам по себе не перезагружает полный конфиг по Clash API (нет такого endpoint); меняется только состояние лаунчера (APIService), чтобы UI и выбор селекторов соответствовали новому config.json.
- `ReloadClashAPIConfig()` уже вызывается при старте sing-box (`process_service.startSingBox`); здесь добавляется вызов после успешного Update (в т.ч. при сохранении из визарда).
