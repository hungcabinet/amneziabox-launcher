# Спецификация: Ошибка Ping — tooltip вместо отдельного окна

## Проблема

На вкладке **Clash API** при нажатии на кнопку **Ping** у прокси выполняется замер задержки. При ошибке (сеть, таймаут, недоступность API и т.п.):
- текст кнопки меняется на **"Error"**;
- в статусной строке показывается «Delay error: …»;
- **отдельно открывается модальное окно** с текстом ошибки (`ShowError`).

Всплывающее окно перебивает работу пользователя и избыточно для быстрого просмотра ошибки по одному прокси.

## Цель

Показывать ошибку пинга **только при наведении** на кнопку, которая в состоянии **"Error"**: tooltip с текстом ошибки. Отдельное окно с ошибкой при неуспешном Ping **не показывать**.

## Требования

1. **При ошибке Ping:**
   - текст кнопки — **"Error"** (без изменений);
   - статусная строка может по-прежнему показывать «Delay error: …» (опционально);
   - **не вызывать** `ShowError` / диалог с ошибкой.

2. **Хранение последней ошибки:**
   - для каждого прокси (по имени) хранить последний текст ошибки пинга;
   - при успешном пинге — очищать сохранённую ошибку для этого прокси.

3. **Tooltip при наведении:**
   - когда кнопка отображает **"Error"**, при наведении мыши на кнопку показывать **tooltip** с текстом последней ошибки для этого прокси;
   - реализация — кнопка с поддержкой tooltip (fyne-tooltip); текст ошибки задаётся сразу при установке «Error» и дублируется в `updateItem` при отрисовке строки.

4. **Тексты в UI** — только английский (constitution).

## Критерии приёмки

- [x] При ошибке Ping не открывается отдельное окно с ошибкой.
- [x] Текст кнопки при ошибке — "Error"; при наведении на такую кнопку показывается tooltip с текстом ошибки.
- [x] При успешном повторном пинге ошибка очищается, tooltip при наведении пустой или не показывается.
- [x] Сборка и тесты (где применимо) проходят; линтер без новых ошибок.

## Поведение (кратко)

| Событие              | Результат                                                                 |
|----------------------|---------------------------------------------------------------------------|
| Ping успешен         | Кнопка: «N ms»; сохранённая ошибка для прокси очищается.                  |
| Ping ошибка          | Кнопка: «Error»; ошибка сохраняется; отдельное окно не показывается.       |
| Наведение на «Error» | Tooltip с текстом последней ошибки для этого прокси.                      |
| Наведение на «Ping»/«N ms» | Tooltip пустой или не показывается (getter возвращает пустую строку). |

## Зависимости

- **Tooltip:** пакет [github.com/dweymouth/fyne-tooltip](https://pkg.go.dev/github.com/dweymouth/fyne-tooltip) — кнопка с `SetToolTip(string)`; слой тултипов в главном окне через `AddWindowToolTipLayer(content, canvas)`.
- **Список прокси** — переиспользуемые ячейки; в `updateItem` задаётся `SetToolTip(GetLastPingError(proxyName))`.
- **Сохранение состояния кнопки:** результат пинга (Delay или ошибка) записывается в список прокси (`ProxyInfo.Delay`: >0 = ms, -1 = error). При обновлении списка из API (в т.ч. при возврате на вкладку Servers) локальные Delay из старого списка переносятся в новый, чтобы надпись на кнопке не сбрасывалась.
