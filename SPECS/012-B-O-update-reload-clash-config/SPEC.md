# Спецификация: сохранение в визарде — только сохранение файлов и Update

Поведение при сохранении в визарде должно быть строго определённым: сохранение файлов и вызов **Update**; без какого-либо дополнительного поведения (в частности, без перезапуска). Перечитывание настроек Clash API из `config.json` выполняется только при запуске sing-box (`ReloadClashAPIConfig` в process_service).

## Проблема

1. **Сохранение в визарде:** при сохранении в визарде допустимо только: запись файлов и вызов Update. Не должно быть дополнительного поведения (перезапуск sing-box, перезапуск приложения и т.п.).
2. **После Update:** по нажатию **Update** выполняется только `RunParserProcess()`: подтягиваются подписки, пересобираются outbounds, записывается `config.json`. Состояние лаунчера (вкладка Servers, Clash API) перечитывается из конфига только при запуске sing-box — это достаточная логика, без вызова Reload после Update.

## Текущее поведение (как сейчас)

При сохранении в визарде (`ui/wizard/presentation/presenter_save.go`):

1. **Сохранение файлов:** сборка конфига из модели → запись `config.json` → валидация sing-box → запись `state.json`.
2. **Обновление статуса в Core:** вызов `UpdateConfigStatusFunc()`.
3. **Дополнительное поведение — перезапуск sing-box:** в `saveStateAndShowSuccessDialog()` (стр. 254–281) при запущенном sing-box выполняется `StopSingBoxProcess()`, затем в горутине после паузы 500 ms — ожидание остановки (до 2 с) и `StartSingBoxProcess(true)`.
4. **Update:** в `completeSaveOperation()` в фоне вызывается `go core.RunParserProcess()` — тот же поток, что и кнопка Update на главной вкладке.

Итого: сейчас делаются сохранение файлов и вызов Update, но дополнительно выполняется перезапуск sing-box при его запущенном состоянии. По контракту перезапуск не должен входить в сохранение в визарде.

## Цель

1. **Визард:** при сохранении в визарде — только сохранение файлов и вызов Update; никакого дополнительного поведения (в т.ч. перезапуска). Перечитывание Clash API из `config.json` — только при запуске sing-box.

## Требования

1. **Сохранение в визарде:** при сохранении в визарде выполнять только сохранение файлов и вызов Update (тот же поток, что и кнопка Update). Не выполнять перезапуск sing-box, перезапуск приложения или иное дополнительное поведение.
2. **Порядок при записи config.json:** сначала валидация конфига через `sing-box check` по временному файлу (в той же директории, имя `config-check.json`), при успехе — бэкап и запись в `config.json`. При ошибке валидации пользователь видит ошибку, рабочий конфиг не перезаписывается; временный файл удаляется.

## Критерии приёмки

- При сохранении в визарде выполняются только сохранение файлов и вызов Update; перезапуск и прочее дополнительное поведение отсутствуют.
- Запись в config.json выполняется только после успешной валидации по временному файлу config-check.json.
- Существующее поведение при ошибке Update не меняется.

## Замечания

- При сохранении в визарде контракт: только запись файлов и вызов Update; перезапуск не входит в контракт и не должен выполняться.
- `ReloadClashAPIConfig()` вызывается только при старте sing-box (`process_service.startSingBox`); вызов после Update не нужен.
