## ⚠️ Известные ограничения и проблемы

### 1. FileServiceAdapter и build tag `cgo`

**Проблема:** `FileServiceAdapter` находится в файле с build tag `//go:build cgo`, что может вызывать ошибки линтера при проверке без CGO.

**Решение:** При реальной компиляции с CGO (как в проекте) всё работает корректно. Ошибки линтера можно игнорировать.

**Статус:** Не критично, работает при сборке проекта

### 2. Отслеживание изменений (решено)

**Проблема:** Флаг `hasChanges` устанавливался при инициализации визарда из-за срабатывания callback'ов виджетов (`outboundSelect`, `finalSelect`, `checkbox`) при создании и обновлении через `SetSelected()`.

**Решение:** 
- Добавлен флаг `UpdatingOutboundOptions` для предотвращения установки `hasChanges` во время программных обновлений
- Флаг `UpdatingOutboundOptions` устанавливается в `true` перед созданием виджетов и сбрасывается в `false` после всех `SetSelected()` вызовов в `RefreshOutboundOptions()`
- Флаг `hasChanges` сбрасывается в `false` внутри того же `UpdateUI()`, что и `RefreshOutboundOptions()`, сразу после сброса `UpdatingOutboundOptions`
- Это гарантирует, что все callback'и, которые могли сработать при инициализации, отработают до сброса `hasChanges`

**Статус:** ✅ Решено (2026-02-09)

### 3. Отслеживание изменений правил

**Проблема:** Изменения правил (добавление, редактирование, удаление) не отслеживаются автоматически.

**Решение:** Добавлены вызовы `MarkAsChanged()` в местах, где изменяются правила:
- В callback'ах `outboundSelect` и `checkbox` для selectable rules (только если `!UpdatingOutboundOptions`)
- В callback'ах `outboundSelect` и `checkbox` для custom rules (только если `!UpdatingOutboundOptions`)
- В callback'е `finalSelect` (только если `!UpdatingOutboundOptions`)
- В диалоге добавления/редактирования правил (`add_rule_dialog.go`)

**Статус:** ✅ Реализовано

### 4. Генерация `experimental.clash_api.secret`

**Проблема:** `experimental.clash_api.secret` генерируется при сохранении конфига через `SaveConfigWithBackup`, но не сохраняется в `config_params` при создании состояния.

**Решение:** Согласно спецификации, `experimental.clash_api.secret` генерируется автоматически при первом сохранении конфига и затем сохраняется в `state.json`. Требуется интеграция с `SaveConfigWithBackup`.

**Статус:** Требует доработки
