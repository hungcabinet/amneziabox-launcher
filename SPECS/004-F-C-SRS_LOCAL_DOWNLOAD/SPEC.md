# Спецификация: Локальное скачивание SRS

## 1. Проблема

### 1.1 Текущее поведение

В `bin/wizard_template.json` используются rule-set с `type: "remote"`, которые загружаются sing-box во время работы по URL вида:
```
https://raw.githubusercontent.com/runetfreedom/russia-v2ray-rules-dat/release/sing-box/rule-set-*/...
```

Для загрузки этих SRS используется outbound `go-any-way-githubusercontent` с `download_detour`, который:
- Является urltest, проверяющим доступность `https://raw.githubusercontent.com/github/gitignore/main/Global/GPG.gitignore`
- Использует только `addOutbounds: ["direct-out"]` — т.е. прямое соединение без прокси

### 1.2 Почему первый запуск VPN невозможен в РФ

1. Домен `raw.githubusercontent.com` часто заблокирован в России.
2. При первом запуске sing-box пытается скачать SRS по remote URL.
3. `go-any-way-githubusercontent` использует direct-out — т.е. прямое соединение.
4. Прямое соединение к заблокированному GitHub не работает.
5. Загрузка SRS не удаётся → sing-box не может корректно инициализироваться → VPN не запускается.

Возникает «замкнутый круг»: VPN нужен для доступа к GitHub, а SRS для работы VPN загружаются с GitHub.

### 1.3 Правила, использующие remote SRS с raw.githubusercontent.com

| Правило | rule_set tag(s) | default |
|---------|-----------------|---------|
| Block Ads (ads-all, soft) | ads-all | true |
| Russia-only services | ru-inside | false |
| Russian blocked resources | ru-blocked-main, ru-blocked-community | false |
| Israel-specific sites | israel-sites | false |
| Games direct | games | true |

По умолчанию включены **Block Ads** и **Games direct** — оба используют remote SRS.

## 2. Требования

### 2.1 Outbound go-any-way-githubusercontent больше не нужен

**Полностью удалить** `go-any-way-githubusercontent` из шаблона и кода. Он не используется никогда — sing-box больше не загружает SRS по remote URL.

### 2.2 Скачивание SRS — кнопка на Rules tab

Для правил, требующих SRS, рядом с иконкой `?` (help) добавляется **маленькая кнопка** с состояниями:

| Состояние | Иконка | Описание |
|-----------|--------|----------|
| Надо скачать | `⬇` | SRS не загружен. По нажатию — запуск скачивания. |
| Идёт скачивание | `🔄` | Загрузка в процессе. Кнопка неактивна. |
| Скачано | `✔️` | SRS загружен. Повторное нажатие — перезагрузка. |

**Логика:**
- **Чекбокс правила заблокирован**, пока SRS не скачан. При попытке включить — попап с сообщением (например: «Сначала загрузите SRS, нажав кнопку ⬇»).
- Кнопка `⬇` — запускает скачивание; при успехе переходит в `✔️`, чекбокс разблокируется.
- Кнопка `✔️` — повторное нажатие запускает перезагрузку SRS (обновление).

### 2.3 Правила по умолчанию (default: true)

Для правил с `default: true`, требующих SRS (Block Ads, Games direct):
- При первом открытии визарда: если SRS ещё нет — автоматически запускаем скачивание (кнопка в состоянии `🔄`); при успехе включаем правило по умолчанию и показываем `✔️`, при ошибке — оставляем выключенным и показываем `⬇`.

### 2.4 Локальное использование SRS

- Файлы сохраняются в `{ExecDir}/bin/rule-sets/{tag}.srs`.
- В конфиге rule_set указываются как `type: "local"`, `format: "binary"`, `path: "<локальный путь>"`.
- sing-box не обращается к raw.githubusercontent.com при работе.

### 2.5 Передача конфигурации

- Конфиг + папка `bin/rule-sets/` могут передаваться другому пользователю — конфиг работает без доступа к GitHub.

### 2.6 Критерии приёмки

1. **go-any-way-githubusercontent удалён** — из wizard_template.json, get_free.json, loader, source_tab, ParserConfig.md.
2. **Правила с SRS** — кнопка ⬇/🔄/✔️ после `?`; чекбокс заблокирован до скачивания.
3. **Скачивание** — по клику ⬇ или ✔️; при ошибке — диалог, правило не включается.
4. **Конфиг** — rule_set с `type: "local"`, `path`; sing-box не обращается к raw.githubusercontent.com.
5. **Первый запуск** — VPN стартует без доступа к GitHub (если правила с SRS выключены или SRS уже скачаны).
6. **Default-правила** — при первом открытии визарда авто-скачивание; при ошибке — правило выключено.

## 3. Структура данных

### 3.1 Rule_set, требующие загрузки

Rule_set считается «требующим SRS», если:
- `type == "remote"`
- `url` содержит `raw.githubusercontent.com`

Правило можно включить **только если** все его SRS уже скачаны локально.

### 3.2 Директория для локальных SRS

Путь: `{ExecDir}/bin/rule-sets/`

Имена файлов: `{tag}.srs`, например:
- `ads-all.srs`, `ru-inside.srs`, `ru-blocked-main.srs`, `ru-blocked-community.srs`, `israel-sites.srs`, `games.srs`

### 3.3 Правило с несколькими SRS

Правило «Russian blocked resources» имеет 2 rule_set: `ru-blocked-main`, `ru-blocked-community`. Одна кнопка на правило:
- Состояние `✔️` — только когда **все** SRS правила скачаны.
- По клику ⬇ — скачиваются **все** отсутствующие SRS (последовательно или параллельно).
- При ошибке одного — показываем диалог, состояние остаётся `⬇`.

### 3.4 Формат rule_set в wizard_template.json

Для правил с SRS в `selectable_rules` оставить в `rule_set`:
- `tag`, `type: "remote"`, `format: "binary"`, `url`
- **Удалить** `download_detour`, `update_interval` (не используются).

При генерации конфига: заменить на `type: "local"`, `format: "binary"`, `path`.

## 4. UI: кнопка SRS и включение правила (Rules tab)

**Расположение:** кнопка размещается после иконки `?` (help) в строке правила.

**Состояния кнопки:**

| Состояние | Отображение | Поведение |
|-----------|--------------|-----------|
| SRS отсутствует | `⬇` | Клик → запуск скачивания |
| Скачивание | `🔄` | Кнопка неактивна, идёт загрузка |
| Скачано | `✔️` | Клик → перезагрузка (обновление) |

**Чекбокс правила:**
- Заблокирован, если SRS не скачан.
- При клике по заблокированному чекбоксу — попап: «Сначала загрузите SRS, нажав кнопку ⬇».
- Разблокируется после успешного скачивания.

**При первом открытии** визарда для правил с `default: true`, требующих SRS:
- Автоматически запускаем скачивание (кнопка `🔄`).
- Успех → включаем правило, кнопка `✔️`.
- Ошибка → правило выключено, кнопка `⬇`.

## 5. Edge cases

| Ситуация | Поведение |
|----------|-----------|
| Закрытие визарда во время скачивания | Скачивание прерывается (context cancel). Частичный файл не используется. |
| Пользователь удалил bin/rule-sets/ вручную | При следующем открытии: файлов нет → кнопка ⬇, чекбокс заблокирован. |
| Диск заполнен при сохранении | Ошибка записи → диалог, правило не включается. |
| 404 или изменён URL | Диалог с ошибкой, кнопка остаётся ⬇. |
| Default-правила: оба (Block Ads, Games) требуют SRS | Скачивание последовательно или параллельно; каждое правило независимо (одно успешно — включено, другое ошибка — выключено). |

## 6. Тексты UI (для реализации)

- Попап при клике по заблокированному чекбоксу: «Сначала загрузите SRS, нажав кнопку ⬇».
- Диалог ошибки скачивания: «Не удалось загрузить SRS: {error}. Проверьте подключение к интернету или используйте VPN.»

---

## 7. Обратная совместимость

- При сохранении через визард конфиг пересобирается — remote rule_set заменяются на local. Это ожидаемое поведение.
- Существующие конфиги с `go-any-way-githubusercontent` при следующем открытии визарда получат обновлённый ParserConfig без этого outbound (он удалён из шаблона).

---

## 8. Риски и ограничения

1. **Размер SRS**: Файлы могут занимать несколько мегабайт.
2. **GitHub заблокирован**: Пользователь не сможет включить правила, требующие SRS, пока нет доступа к raw.githubusercontent.com. Решение: использовать VPN/прокси вне приложения или получить rule-sets от другого пользователя.
3. **Права на запись**: Директория `bin/rule-sets/` должна быть доступна для записи.
