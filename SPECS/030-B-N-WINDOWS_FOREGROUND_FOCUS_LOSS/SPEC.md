# Спецификация: потеря фокуса ввода в сторонних приложениях при работающем лаунчере (Windows)

Задача — **найти причину** (или узкое множество причин) и зафиксировать воспроизведение; **исправление** — отдельный этап после подтверждённой гипотезы (может быть вынесено в follow-up SPEC при необходимости).

## Платформа

- **Только Windows** (наблюдения на Win 10 22H2; при проверке фиксировать точную сборку).
- macOS/Linux — вне скоупа, если не появятся аналогичные отчёты.

## Проблема

При **запущенном** Singbox Launcher в фоне (окно открыто или свернуто в трей — без разницы) у пользователя **периодически пропадает фокус ввода** в других приложениях (браузер, мессенджеры и т.д.): перестаёт печататься текст в поле, пока снова не кликнуть в поле. **После выхода из лаунчера** симптом исчезает.

### Наблюдения от пользователя (референс)

- Интервал: **примерно 10–30 с** (не фиксированный).
- **sing-box не запускали** — поведение то же.
- **Уведомления Windows** для приложения/в целом отключали — **фокус всё равно слетает**.
- Окружение: ноутбук, **два монитора**; билд лаунчера **0.8.44-gb57dabe-prerelease**.

## Цель

1. **Воспроизвести** на контрольной машине или подтвердить по логам **корреляцию** слёта фокуса с конкретными вызовами в процессе лаунчера (UI, трей, Win32).
2. **Документировать** наиболее правдоподобную причину и список отсечённых гипотез.
3. **Предложить** минимальное исправление или смягчение (отдельный PR), если корневая причина в нашем коде; если в Fyne/ОС — зафиксировать в отчёте и при наличии — обходной путь.

## Не в скоупе (пока)

- Изменение поведения прокси/TUN без связи с доказанным сценарием фокуса.
- Масштабный рефакторинг GUI.

## Гипотезы (для проверки)

1. **Периодические обновления GUI** — в частности цикл автообновления блока версии sing-box на вкладке Core (`startAutoUpdate` в `ui/core_dashboard_tab.go`: первые интервалы ~20–35 с, затем реже) и связанные **`fyne.Do`**.
2. **Обновление system tray** — `SetSystemTrayIcon` / `SetSystemTrayMenu` без проверки «изменилось ли»; частые пересборки меню при Clash API (у части сценариев без ядра слабее).
3. **Вызовы активации окна** — `Show` / `RequestFocus` из фоновых путей (маловероятно при «без действий пользователя», но проверить grep’ом и трассировкой).
4. **Fyne / драйвер / мультимонитор** — внешний фактор; контрольный тест с одним монитором и обновлением Fyne.
5. **Системные уведомления** — для данного репорта **отсекаются** наблюдением пользователя; оставить в отчёте как проверенное.
