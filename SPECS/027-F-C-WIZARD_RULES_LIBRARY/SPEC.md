# Спецификация: Rules — один список custom, библиотека из шаблона

## 1. Проблема

Сейчас правила маршрута размазаны по двум слоям: **selectable** из шаблона + отдельное состояние + **custom_rules**. Merge жёсткий, порядок в UI не совпадает с желанием иметь одну очередь правил.

## 2. Цель

Один источник для **route**: массив `**custom_rules`**. Запись может появиться из **начального засева** (только при первом появлении визарда без сохранённого state), из **миграции** старого формата, **вручную** (Add rule) или **из библиотеки** (Add from library). Дальше с любой записью одни и те же действия: вкл/выкл, outbound, ↑↓, Edit, Delete.

**Библиотека** — все объекты `**selectable_rules`** в текущем шаблоне: только для показа в модалке и копирования в конец `custom_rules`. Изменения шаблона **не** пересобирают уже сохранённый список правил.

## 3. Правила поведения


| ID  | Требование                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| R1  | UI на английском (CONSTITUTION).                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| R2  | На вкладке **Rules** кнопка **Add from library** рядом с добавлением ручного правила. По нажатию — модальное окно: прокручиваемый список, у каждой строки чекбокс, **label**; **description** — tooltip или вторая строка, если уместно. Внизу **Cancel** и **Add selected**.                                                                                                                                                                                      |
| R3  | **Add selected:** для каждой отмеченной строки — **глубокая копия** пресета в **конец** `custom_rules` (`rule` / `rules` / `rule_sets` и всё нужное для конфига и Edit). **Дедупликации нет** — один и тот же пресет можно добавить несколько раз.                                                                                                                                                                                                                 |
| R4  | **Первый запуск без сохранённого state визарда:** в `custom_rules` попадают только пресеты из `selectable_rules`, у которых в JSON `**"default": true`**, **в порядке следования в шаблоне**. Enabled / outbound — по полям пресета и текущей логике нового custom-правила. После появления state поле `**default`** в шаблоне на уже сохранённые строки **не** влияет.                                                                                            |
| R5  | **Миграция (один раз):** если загружен старый state с отдельным слоем selectable — перенести в один список `custom_rules`: сначала блок из пресетов шаблона в порядке шаблона с переносом **enabled** и **selected_outbound** из `selectable_rule_states`, затем прежние `custom_rules` без изменения их относительного порядка. Выставить признак «миграция выполнена», дальше selectable **не** участвует в merge и в UI как отдельный блок. Подробности — PLAN. |
| R6  | `**MergeRouteSection`:** после миграции **пользовательские** вставки в `route` строятся **только** из `custom_rules` (порядок массива); **скелет** `route` из шаблона (базовые `rules` / `rule_set`, не дублирующие пресеты) сохраняется как сейчас.                                                                                                                                                                                                                     |
| R7  | Пресеты с **rule_set (SRS):** при копировании — та же семантика, что у обычных custom с SRS.                                                                                                                                                                                                                                                                                                                                                                       |
| R8  | Если список `custom_rules` пуст — на вкладке Rules **empty state** и явный вход в библиотеку (кнопка / ссылка на тот же диалог).                                                                                                                                                                                                                                                                                                                                   |
| R9  | Документация: **docs/WIZARD_STATE.md**, **docs/ARCHITECTURE.md** — один список, библиотека, миграция, поле `**default`** только для первого засева.                                                                                                                                                                                                                                                                                                                |


## 4. Критерии приёмки

1. Сохранённый профиль: правила и **route** соответствуют `custom_rules`; шаблон меняется — сохранённые правила сами не пересобираются.
2. Новый профиль без state: в списке только пресеты с `**"default": true`**, порядок как в шаблоне.
3. Add from library добавляет выбранные копии в конец; повторный выбор того же пресета даёт ещё одну копию.
4. После миграции старого state нет двойного учёта selectable в **route**; блок selectable на Rules убран.
5. Empty state при пустом списке.
6. Доки обновлены; **go build / test / vet** по **IMPLEMENTATION_PROMPT**.
