## Причина закрытия

От реализации типа правила «SRS» в рамках **отдельной** задачи 014 отказались. Функциональность типа **SRS** (константа `srs`, выбор rule-set'ов по URL, каталог runetfreedom, генерация tag и rule_set) перенесена в объединяющую задачу **018-F-C-CUSTOM_RULE_SUBSYSTEM_REFACTOR**.

В SPEC 018 тип SRS входит в общий рефакторинг подсистемы Custom Rule (константы типов `ips`/`urls`/`processes`/`srs`/`raw`, вкладка Raw, поле `params`, документация по state). Детальное описание UI, каталога runetfreedom и структуры данных для типа SRS по-прежнему задаётся **спецификацией 014** (SPEC.md) и используется при реализации 018 как референс.

## Связь с задачей 018

- **SPECS/018-F-C-CUSTOM_RULE_SUBSYSTEM_REFACTOR** — разделы 1.4, 4.1, 4.3: тип `srs`, вкладка Form с выбором rule-set'ов.
- Спека 018 ссылается на **SPECS/014-F-C-RULE_TYPE_SRS_URL** (после переименования папки) для полного описания типа SRS.
