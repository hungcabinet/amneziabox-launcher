# План: macOS — Application Support / Logs и изменяемый Bundle ID

## 1. Архитектура

### 1.1 Разделение корня данных

- Ввести явное понятие **«data root»** для файловых операций лаунчера:
  - **macOS + .app:** `dataRoot` = базовый каталог под `~/Library/Application Support/<bundleID>/` (точная схема подкаталогов — см. ниже).
  - **иначе (macOS без .app, Windows, Linux):** `dataRoot` = `ExecDir` (текущее поведение).

### 1.2 Логи

- **macOS + .app:** корень логов = `~/Library/Logs/<bundleID>/` (или подкаталог с тем же именем, что и у приложения — не смешивать с `bin/` без необходимости).
- **иначе:** как сейчас — `ExecDir/logs/`.

### 1.3 Определение «запущены из .app»

- Реализовать в `internal/platform` (или рядом) функцию вида `IsMacOSAppBundle() bool`:
  - `runtime.GOOS == "darwin"` и путь к исполняемому файлу содержит **`.app/Contents/MacOS/`** (нормализовать через `filepath.Clean` / разбор пути).
- Покрыть граничные случаи: симлинки (по возможности разрешать `filepath.EvalSymlinks` только если безопасно и недорого).

### 1.4 Bundle ID в коде и сборке

- Добавить в **`internal/constants`** (или отдельный маленький пакет) строку **`MacOSBundleID`** с дефолтом `com.singbox-launcher`, перезаписываемую **`ldflags -X`** при релизной сборке.
- В **`build/build_darwin.sh`**:
  - переменная окружения, например **`MACOS_BUNDLE_ID`**, по умолчанию `com.singbox-launcher`;
  - подстановка в **`CFBundleIdentifier`** при генерации `Info.plist`;
  - передача того же значения в **`go build`** через `-ldflags "-X singbox-launcher/internal/constants.MacOSBundleID=..."` (путь к символу уточнить по фактическому имени пакета/переменной).
- **CI** (`.github/workflows`): при вызове `build_darwin.sh` не задавать переменную — остаётся дефолт; при необходимости кастомной сборки — задокументировать в README.

### 1.5 Интеграция с `FileService`

- **`NewFileService`** (или последующий шаг инициализации):
  - вычислить `execDir` как сейчас;
  - вычислить `dataRoot` и `logsRoot` по правилам выше;
  - `GetConfigPath`, `GetBinDir`, `GetRuleSetsDir`, `EnsureDirectories` — от **`dataRoot`**, а не от `execDir`, на macOS в режиме .app;
  - открытие лог-файлов — из **`logsRoot`**.

### 1.6 Миграция со старого portable-пути внутри .app

- Однократная процедура (флаг в коде или версия «миграции» в пустом маркер-файле):
  - если `dataRoot` указывает на Application Support и каталог пуст **или** нет `config.json`, а в `ExecDir/bin/` есть старые файлы — скопировать дерево `bin/` (и при необходимости `logs/`) с логированием.
- Детали и идемпотентность — в `TASKS.md`; избегать дублирования при повторном запуске.

## 3. Риски

- **Права доступа:** запись в `~/Library` у обычного пользователя обычно есть; при страничных настройках — диагностировать через debuglog.
- **Пути в сгенерированном sing-box конфиге:** если где-то зашиваются абсолютные пути к SRS — проверить после смены корня.
- **Тесты:** юнит-тесты для резолва путей без GUI — через table-driven тесты с подменой `GOOS` / тестовыми путями (если вынести чистую функцию).
