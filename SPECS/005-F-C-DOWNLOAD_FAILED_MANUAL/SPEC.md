# Спецификация: единая подсистема «ручная загрузка при ошибке»

## Проблема

При ошибках сети (или иных сбоях загрузки) пользователь не может скачать SRS, Wintun, sing-box или wizard_template.json. Текущее поведение — показ простого диалога с текстом ошибки, без подсказки, что делать дальше.

## Цель

Унифицированная подсистема: при любой неуспешной загрузке перечисленных ресурсов показывать однотипный диалог с:
- коротким сообщением об ошибке и подсказкой «скачайте вручную»;
- ссылкой «Open download page» для ручного скачивания;
- кнопкой «Open folder» для открытия папки, куда положить файл;
- кнопкой «Close».

Одинаковое поведение для всех типов загружаемых файлов.

## Требования

1. **Единая точка отображения** — одна функция (например, `ShowDownloadFailedManual`), вызываемая из всех мест неуспешной загрузки.
2. **Содержимое диалога (все тексты на английском):**
   - заголовок — по контексту (например, «sing-box download failed»);
   - текст: единая фраза «Download failed. See the log for details.» + подсказка «Please download the file manually and place it in the folder below.»;
   - опционально: гиперссылка «Open download page» (открывает URL в браузере) и справа от неё кнопка с иконкой копирования (копирует URL в буфер обмена);
   - опционально: кнопка «Open folder» (открывает целевую папку в проводнике ОС);
   - кнопка «Close». Диалог собирается через `internal/dialogs.NewCustom`.
3. **Охват ресурсов:**
   - **sing-box** — при ошибке `DownloadCore` (Core Dashboard): ссылка на https://github.com/SagerNet/sing-box/releases, папка `bin/`.
   - **wintun.dll** — при ошибке `DownloadWintunDLL`: ссылка на https://www.wintun.net/, папка `bin/`.
   - **wizard_template.json** — при ошибке загрузки шаблона (кнопка в Core Dashboard или отсутствие файла при открытии визарда / при «New» в Load State): ссылка из `GetTemplateURL()`, папка `bin/`.
   - **SRS (rule-set файлы)** — при ошибке `DownloadSRS` в визарде: ссылка на первый URL из правила, папка `bin/rule-sets/`.
4. **Реализация** — диалог в `internal/dialogs`: функция `NewCustom` (перенесена из `ui/components`) и `ShowDownloadFailedManual`, строящая контент и вызывающая `NewCustom`. Без импорта `ui/components` (избежание цикла с `core`).

## Критерии приёмки

- [ ] При ошибке загрузки sing-box, wintun, шаблона или SRS показывается диалог с ссылкой (и кнопкой копирования ссылки), а также кнопкой открытия папки.
- [ ] Тексты в диалоге только на английском.
- [ ] Сборка и тесты (где применимо) проходят; линтер без новых ошибок.
- [ ] Ошибки пользователю по-прежнему показываются только через единую точку (dialogs); логирование — через `debuglog`.

## Структуры данных

- **ShowDownloadFailedManual**: параметры `window`, `title`, `downloadURL`, `targetDir` (message — фиксированные константы в коде).
- **NewCustom**: `title`, `mainContent`, `buttons`, `dismissText`, `parent` — общий диалог с Border (контент сверху, кнопки снизу), ESC закрывает при непустом dismissText.
- Константы URL: `internal/constants` (SingboxReleasesURL, WintunHomeURL).
