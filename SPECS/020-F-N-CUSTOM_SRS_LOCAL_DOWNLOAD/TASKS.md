- [ ] Найти текущую реализацию скачивания SRS и кнопки `⬇/🔄/✔️` для встроенных правил (по задачам `004-F-C-SRS_LOCAL_DOWNLOAD`, `005-F-C-DOWNLOAD_FAILED_MANUAL`).
- [ ] Вынести/обобщить функции работы с `{ExecDir}/bin/rule-sets/`:
  - [ ] Helper для проверки наличия локального файла по tag (`HasLocalSRS` или аналог);
  - [ ] Helper для скачивания набора `(tag, url)` с сохранением в `{tag}.srs` и корректной обработкой отмены/ошибок.
- [ ] Добавить helper для генерации конфигурации rule_set (`local`/`remote`) на основе наличия локального файла и remote URL.

## Этап 2: Интеграция с пользовательскими SRS в моделях

- [ ] Убедиться, что в моделях визарда для пользовательских SRS-правил (type `srs`) доступен список `(tag, url)`:
  - [ ] Проверить структуры `PersistedCustomRule`, `RuleState` и связанные типы в `ui/wizard/models/wizard_state_file.go` и соседних файлах.
  - [ ] При необходимости добавить helper-метод/структуру, не меняя формат state.

## Этап 3: UI вкладки Rules — кнопка SRS для пользовательских правил

- [ ] В `ui/wizard/tabs/rules_tab.go`:
  - [ ] Найти текущий код, который рисует кнопку SRS для встроенных правил.
  - [ ] Расширить построение списка правил так, чтобы:
    - [ ] пользовательские правила типа `srs` также получали кнопку SRS;
    - [ ] состояние кнопки (`⬇/🔄/✔️`) определялось через общий helper и наличие всех `{tag}.srs` для правила;
    - [ ] клик по `⬇` запускал скачивание недостающих SRS этого правила;
    - [ ] клик по `✔️` запускал принудительную перезагрузку всех SRS правила.
- [ ] Реализовать блокировку чекбокса пользовательского SRS-правила до скачивания всех SRS:
  - [ ] заблокировать чекбокс, если хотя бы один tag правила не имеет локального файла;
  - [ ] при клике по заблокированному чекбоксу показывать попап с текстом на английском (по аналогии с 004).

## Этап 4: Авто-скачивание при первом открытии Rules

- [ ] В `rules_tab.go` (или соответствующем контроллере визарда):
  - [ ] Добавить механизм одноразовой инициализации при первом открытии вкладки Rules.
  - [ ] Для включённых пользовательских SRS-правил:
    - [ ] при отсутствии локальных SRS автоматически запускать скачивание (состояние `🔄`);
    - [ ] при успехе оставлять правило включённым и переводить кнопку в `✔️`;
    - [ ] при ошибке — отключать правило, переводить кнопку в `⬇` и показывать диалог ошибки.

## Этап 5: Генерация конфига

- [ ] В `ui/wizard/business/create_config.go`:
  - [ ] Найти текущую генерацию rule_set для встроенных SRS и пользовательских SRS (из 014/018).
  - [ ] Подключить общий helper `BuildRuleSetConfig` (или аналог), который:
    - [ ] проверяет наличие `{ExecDir}/bin/rule-sets/{tag}.srs`;
    - [ ] возвращает конфиг с `type: "local"/"remote"`, `path` или `url`.
  - [ ] Убедиться, что:
    - [ ] встроенные правила с SRS используют `local`/`remote` по тем же правилам, что и до изменений;
    - [ ] пользовательские SRS-правила автоматически используют `local` при наличии локальных файлов и `remote` при их отсутствии.
